# 云打包工具需求整理

本文档汇总本项目开发过程中，用户提出的全部需求与补充说明，按时间顺序整理。

## 初始需求

1. 从 0 开发一个前端网页项目云打包工具。
2. 使用 `Node.js + Vue`。
3. 使用 `JSON` 保存用户和项目配置信息。
4. 项目具有以下功能：
   - 登录
   - 用户管理
   - 项目列表
5. 每个人能看的项目需要在用户管理里面配置。
   - 一般是先在项目列表创建项目，再去用户管理勾选用户权限。
   - 只有 `admin` 账户可以看到用户管理列表。
6. 项目列表创建项目字段有：
   - `id`
   - 项目名称
   - 脚本
   - 创建时间
   - 更新时间
7. 用户管理字段有：
   - 账户
   - 密码
   - 项目权限

## 交互问题修复

1. 创建用户时，账户输入一个字母会瞬间失去焦点，之后不能继续输入。
2. 需要修复该问题。

## 权限规则补充

1. 用户创建项目成功时，这个项目的权限默认给到这个用户。
2. 所有人都能创建项目，但用户管理只有 `admin` 可以。
3. 所有人都有项目列表的所有操作权限。
   - 只要他能看到项目，就能执行该项目的所有操作。
   - 项目权限是在用户管理里面控制的。
   - `admin` 天然具有所有权限。
4. 如果用户没有某个项目的权限：
   - 不只是不能看到该项目。
   - 即使他自己盲猜 `ID` 去调项目列表的删改查接口，也必须不允许。

## 项目执行功能需求

1. 开始丰富项目列表功能。
2. 用户脚本往往是：
   - `git clone` 某个项目
   - 进入项目根目录
   - 执行仓库里的 `deploy.sh`
   - 执行 `build` 和产物提交
3. 这些脚本内容由用户自己编写，系统不用干涉脚本内容本身。
4. 用户点击执行时，需要有：
   - 正在执行的状态
   - 进度条
   - 执行详情
5. 执行详情必须打印执行脚本产生的所有日志。
6. 日志展示使用黑色主题。
7. 正在执行中的项目不允许再次点击执行。
8. 每个项目需要有执行记录，包括：
   - 成功状态
   - 错误状态
   - 历史日志
9. 所有脚本务必在 `.build` 目录下执行。
10. 执行完毕之后务必删除 `.build` 下的临时文件。
11. 临时文件名可采用：
   - `项目id + 项目名称 + 时间`

## UI 优化需求

1. 美化 UI。
2. 把项目列表的创建项目和详情收进弹窗。
3. 当前列表字段挤在一起，需要优化展示。
4. 把操作列右侧固定。
5. 后续补充：
   - 先隐藏 `ID`
   - 隐藏脚本摘要
   - 在这种布局下，操作列是否固定可自行判断处理

## 项目字段扩展需求

1. 给项目列表表设计加一个 `类型` 字段。
   - 分为 `线上环境` 和 `测试环境` 两种类型。
   - 需要在列表高亮展示。
   - 在项目列表 UI 上加一个 Tab。
   - 切换 `测试` 和 `线上` 两个 Tab 进行筛选。
2. 给项目列表表设计加一个 `Node.js 版本` 字段。
   - 需要写死当前 `16+` 的 `Node.js LTS` 版本。
   - 供用户选择。
   - 可以编辑。
   - 后续补充备选版本：
     - `16.16.0`
     - `18.16.0`
     - `22.22.0`
     - `24.14.0`
3. 给项目列表表设计加一个 `包管理工具` 字段。
   - 候选为 `pnpm / npm / yarn`
   - 供用户选择。
   - 可以编辑。
4. 在用户执行项目时：
   - 需要判断当前环境是否有对应的 `Node.js` 版本和包管理工具。
   - 没有就自动安装。
   - 部署环境默认配有 `nvm` 工具，可以使用。
5. 由于字段变多，可以考虑继续固定操作列。

## 辅助要求

1. 使用 `pnpm` 安装依赖。
2. 将所有用户提示词整理进项目中的一个 Markdown 文件。

## 后续新增需求

1. 手动停止执行。
2. 执行结果摘要，除了全量日志，还需要提炼：
   - 开始时间
   - 结束时间
   - 耗时
   - 退出码
   - 最后一段错误
3. 操作审计作为一个独立菜单，只允许 `admin` 查看。
4. 最近执行人和最近执行结果直接展示在项目列表里。
5. 历史保留策略：
   - 日志一直存 JSON 迟早会越来越大
   - 每个项目只保存最新 10 次执行日志
6. 构建服务全局最多并发 5 个项目。
   - 超过后提示：`当前服务已满，请等待其他任务完成`

## 自动执行需求

1. 创建项目和编辑项目时新增自动执行勾选框。
2. 勾选自动执行后，需要填写项目仓库的 SSH 地址。
3. 系统需要根据仓库地址拉取远端所有分支，供用户选择监听分支。
4. 保存后由后端统一调度器轮询，不依赖前端页面是否打开。
5. 轮询规则：
   - 只轮询开启自动执行的项目
   - 每天 `08:00` 到 `22:00` 之间执行
   - 每 `3` 分钟统一扫一轮
   - 其他时间不扫
6. 轮询逻辑：
   - 检查配置分支的最新 commit 是否变化
   - commit 变化后自动触发项目执行
